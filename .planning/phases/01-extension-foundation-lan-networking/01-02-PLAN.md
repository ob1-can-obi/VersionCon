       - Class `AuthHandler` with constructor taking `inviteCode: string`
       - Private `rateLimitMap: Map<string, { attempts: number; lastAttempt: number }>` for per-IP rate limiting
       - Method `validateInviteCode(provided: string): boolean`:
         - Uses `crypto.timingSafeEqual(Buffer.from(provided), Buffer.from(this.storedCode))` for constant-time comparison

       - Method `regenerateCode(): string`:
         - Generates new 6-character code using custom alphabet `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (excludes 0/O/I/1 for readability)
         - Uses `crypto.randomBytes` + modulo mapping to custom alphabet
         - Updates internal stored code
         - Returns new code
       - Getter `currentCode: string`

       - Import `WebSocketServer, WebSocket` from `ws`, import `createServer` from `net`
       - Import `AuthHandler`, `BandwidthMonitor`, protocol functions, session types, event types
       - Class `SessionHost` implements `SessionEventEmitter` (via internal EventEmitter pattern with typed listeners map)
       - Constructor takes `config: SessionConfig, hostDisplayName: string`
       - Private fields: `wss: WebSocketServer | null`, `members: Map<string, { ws: WebSocket; member: Member; isAlive: boolean }>`, `heartbeatInterval: NodeJS.Timeout | null`, `authHandler: AuthHandler`, `bandwidthMonitor: BandwidthMonitor`
       - Method `async start(): Promise<number>`:
         - If `config.port === 0`, find free port via `net.createServer().listen(0)`
         - Create `new WebSocketServer({ port, maxPayload: config.maxPayloadBytes, perMessageDeflate: false })`
         - Register `connection` handler that calls `handleConnection(ws, req)` (req gives client IP from `req.socket.remoteAddress`)
         - Start heartbeat interval (15 seconds)
         - Emit `session-created` event
         - Return actual port number
       - Private `handleConnection(ws: WebSocket, req: IncomingMessage)`:
         - Extract client IP from `req.socket.remoteAddress`
         - Set 10-second auth timeout: if no `auth-request` received, close connection
         - On `message`: parse with `parseMessage()`, route by `msg.type`:
           - `auth-request`: check rate limit via authHandler, validate invite code via authHandler, if accepted: generate member ID via `crypto.randomUUID()`, create Member object, add to members map, send `auth-response` with `accepted: true`, broadcast `member-joined` to all other members, send `state-sync` to new member
           - `auth-request` rejected: send `auth-response` with `accepted: false` and reason, close connection
           - `kick-member`: only allowed from host member (first authenticated member with role 'host'), find target, send `member-kicked`, close target ws
           - `regenerate-invite`: only from host member, call `authHandler.regenerateCode()`, broadcast `invite-regenerated` to all
           - `heartbeat-pong`: mark member as alive
         - On `pong` (ws native): mark member isAlive = true
         - On `close`: remove from members, emit `member-left`, broadcast `member-left` to all
         - Track bandwidth: on each message received call `bandwidthMonitor.recordReceived`
       - Private `broadcast(msg: ProtocolMessage, excludeId?: string)`: send to all authenticated members except excludeId
       - Private `startHeartbeat()`: setInterval 15000ms, for each member: if !isAlive terminate + remove + broadcast leave; else set isAlive=false and ws.ping()
       - Method `kickMember(memberId: string, reason: string): void`
       - Method `getMembers(): Member[]`
       - Method `getBandwidthStats()`: delegates to bandwidthMonitor.getAllStats()
       - Method `regenerateInviteCode(): string`
       - Method `stop(): void`: clear heartbeat interval, close all connections, close wss, emit `session-ended`
       - Method `dispose(): void`: calls stop()

           - If aborted or attempts >= maxAttempts, call onFailed() and return
           - Calculate delay via getReconnectDelay(this.attempts)
           - Set timer, on fire: await connect(), if success reset(), else attempts++ and recurse
         - Method `reset(): void`: attempts = 0, clear timer
         - Method `abort(): void`: aborted = true, clear timer

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Host WebSocket | Untrusted network input from any machine on the LAN; must validate invite code and all message payloads |
| Client auth phase | Unauthenticated connection before invite code is validated; auth timeout required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Spoofing | AuthHandler | mitigate | Constant-time invite code comparison via `crypto.timingSafeEqual`; rate limit 5 attempts/minute per IP |
| T-01-04 | Denial of Service | SessionHost | mitigate | `maxPayload` limit on WebSocket server; 10-second auth timeout for unauthenticated connections; heartbeat terminates dead connections |
| T-01-05 | Tampering | SessionHost message handler | mitigate | All incoming messages validated via `parseMessage()` which rejects malformed JSON and unknown types; handlers wrapped in try/catch |
| T-01-06 | Elevation of Privilege | kick-member / regenerate-invite | mitigate | Only host-role member can execute admin commands; server validates role before processing |
| T-01-07 | Spoofing | Member identity | mitigate | Server assigns member IDs via `crypto.randomUUID()` after auth; display names are cosmetic only |
| T-01-08 | Denial of Service | Large message flood | mitigate | Per-connection bandwidth monitoring; `maxPayload` enforced by ws library at transport level |
</threat_model>
