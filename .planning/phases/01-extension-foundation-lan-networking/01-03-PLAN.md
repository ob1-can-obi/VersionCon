    1. Create `src/utils/network.ts`:
       - Import `os` and `net` from Node.js
       - Export function `getLocalIPv4(): string`:
         - Iterate `os.networkInterfaces()`
         - Return first non-internal IPv4 address
         - Fallback: return `'127.0.0.1'`
       - Export function `getAllIPv4Addresses(): Array<{ name: string; address: string }>`:
         - Iterate all interfaces, collect non-internal IPv4 entries
         - Return array of `{ name: interfaceName, address: ipAddress }`
       - Export function `findFreePort(): Promise<number>`:
         - Create `net.createServer()`, listen on port 0
         - Read assigned port from `server.address()`
         - Close server, resolve with port
         - On error, reject

    2. Create `src/network/discovery.ts`:
       - Import `Bonjour` from `bonjour-service`
       - Constants: `SERVICE_TYPE = 'versioncon'`, `BROWSE_TIMEOUT_MS = 10000`
       - Export interface `DiscoveredSession { name: string; host: string; port: number }`
       - Export class `DiscoveryManager`:
         - Private `bonjour: Bonjour | null = null`
         - Private `publishedService: any | null = null` (bonjour-service Service type)
         - Private `browser: any | null = null`
         - Private `initBonjour(): Bonjour`:
           - Lazily create Bonjour instance (only when needed, since mDNS may not be available)
           - Wrap in try/catch -- if bonjour-service fails to initialize (e.g., no multicast support), log warning and return
         - Method `publishSession(name: string, port: number): boolean`:
           - Try: init bonjour, call `bonjour.publish({ name, type: SERVICE_TYPE, port, txt: { version: '1' } })`
           - Return true on success, false if bonjour unavailable
           - NOTE: This is ADDITIVE per locked decision -- manual IP is the primary path
         - Method `browseSessions(onFound: (session: DiscoveredSession) => void, onTimeout?: () => void): void`:
           - Try: init bonjour, call `bonjour.find({ type: SERVICE_TYPE }, (service) => { onFound({ name: service.name, host: service.host || service.addresses?.[0], port: service.port }) })`
           - Set timeout of BROWSE_TIMEOUT_MS to call onTimeout if provided
           - If bonjour unavailable, immediately call onTimeout
         - Method `stopBrowsing(): void`: stop browser if active
         - Method `unpublish(): void`: stop published service if active
         - Method `dispose(): void`: unpublish, stop browsing, destroy bonjour instance
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mDNS multicast network | mDNS responses come from any device on the local network segment; session names and IPs are untrusted |
| globalState / SecretStorage | Trusted platform storage; data is local to this VS Code instance |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Spoofing | DiscoveryManager | accept | mDNS sessions are convenience hints only; actual auth happens via invite code at connection time. Spoofed mDNS entries lead to auth failure, not compromise. |
| T-01-10 | Information Disclosure | DiscoveryManager | accept | Session name + port visible on LAN via mDNS. This is by design for discoverability. No sensitive data in mDNS TXT records. |
| T-01-11 | Information Disclosure | SessionHistory | mitigate | Session history stores IP + port + display name in globalState (not secrets). No invite codes stored in history -- invite codes go to SecretStorage only. |
| T-01-12 | Tampering | SecretStore | mitigate | Invite codes stored via VS Code SecretStorage API which uses platform-native encryption (Keychain/Credential Manager/Keyring) |
</threat_model>
