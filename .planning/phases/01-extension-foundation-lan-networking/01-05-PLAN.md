## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Join form -> Extension Host | User input (IP, port, invite code, display name) from webview is untrusted; must validate before creating client |
| Sidebar webview -> Extension Host | Sidebar postMessage actions (kick, disconnect) must verify role authorization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-16 | Tampering | JoinPanel input validation | mitigate | Validate hostIp format, port range 1-65535, non-empty inviteCode and displayName before creating SessionClient |
| T-01-17 | Elevation of Privilege | Sidebar kick-member | mitigate | SidebarProvider checks `role === 'host'` before forwarding kick request; member-role users cannot trigger admin actions |
| T-01-18 | Tampering | Webview CSP | mitigate | Nonce-based CSP on join panel and sidebar webviews; `default-src 'none'` prevents script injection |
| T-01-19 | Spoofing | Session history quick-connect | accept | Saved sessions may point to different hosts after IP reassignment. Low risk -- user confirms connection, invite code auth validates identity. |
</threat_model>
