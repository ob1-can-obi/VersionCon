On failure: set error from auth failure reason, isConnecting=false, send state-update.
         - `join-quick-connect`: same as join-connect but pre-fills from SavedSession (asks for inviteCode only since it may have changed)
         - `join-select-discovered`: populate hostIp and port from discovered session
         - `join-remove-history`: remove entry from session history, update state
