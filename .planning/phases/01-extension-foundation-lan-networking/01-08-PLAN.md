In `startClientHeartbeat` (line 296-313), change the ping mechanism to use the HeartbeatManager's built-in pong detection by hooking into native WebSocket pong events instead:

Replace the entire `startClientHeartbeat` method:

```typescript
private startClientHeartbeat(): void {
  // Listen for native WebSocket pong frames from the server's ws.ping()
  if (this.ws) {
    this.ws.on('pong', () => {
      this.heartbeat.receivedPong();
    });
  }

  this.heartbeat.start(
    () => {
      // Send a native WebSocket ping (not protocol-level) to match server's pong handler
      if (this.ws && this.ws.readyState === WebSocket.OPEN) {
        this.ws.ping();
      }
    },
    () => {
      // Dead connection detected — trigger reconnect
      if (this.ws) {
        this.ws.close();
      }
    },
  );
}
```

This changes the client from sending protocol-level `heartbeat-ping` messages (which the server ignores) to native WebSocket `ping()` frames. The server already handles native pong responses (line 241-248 in SessionHost.ts, `ws.on('pong', ...)`). And the client now listens for native `pong` events to call `receivedPong()`.

ALSO keep the existing `heartbeat-ping` protocol case (line 268-276) as-is — it handles the REVERSE direction: when the SERVER sends a protocol-level heartbeat-ping to the client. Although the server currently uses native ping frames, keeping the protocol handler is defensive and costs nothing.

**Fix 1 — Host disconnect confirmation in extension.ts (line 142-159):**

The `sidebarProvider.onDisconnectRequested` callback currently calls `activeHost.stop()` immediately without confirmation. When the user is a host, show a warning dialog first.

Replace the disconnect handler at line 142-159 with:

```typescript
sidebarProvider.onDisconnectRequested(async () => {
  // If host, show confirmation dialog before disconnecting (D-15)
  if (activeHost) {
    const members = activeHost.getMembers();
    // Exclude the host from the count — only count other members
    const otherMemberCount = members.filter(m => m.role !== 'host').length;

    if (otherMemberCount > 0) {
      const choice = await vscode.window.showWarningMessage(
        `Ending this session will disconnect ${otherMemberCount} member${otherMemberCount > 1 ? 's' : ''}. Continue?`,
        { modal: true },
        'End Session',
      );
      if (choice !== 'End Session') {
        return; // User canceled — do nothing
      }
    }

    activeHost.stop();
    activeHost = null;
  }

  if (activeClient) {
    activeClient.disconnect();
    activeClient = null;
  }

  statusBarManager.setStatus('disconnected');
  sidebarProvider.updateState({
    connectionStatus: 'disconnected',
    sessionName: null,
    role: null,
    members: [],
    bandwidthStats: null,
  });
});
```

Note: The handler signature changes from synchronous `() => void` to `async () => Promise<void>`. The `onDisconnectRequested` type in SidebarProvider accepts `() => void`, and an async function satisfies that (returns Promise which is ignored). No type change needed in SidebarProvider since Promise<void> is assignable to void return.
