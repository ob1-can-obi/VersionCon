<critical_constraints>
1. **No dismiss-only escape hatch.** When the user dismisses (Esc / X) the modal block on stage / unstage / debug / task, the action MUST NOT proceed. The only path forward is to click **Sync**.
2. **Modal, not toast.** Use `vscode.window.showInformationMessage(msg, { modal: true }, 'Sync')` (or `showWarningMessage` with modal). The single button is **Sync**. There is no "Ignore", no "Mark Synced", no "Continue anyway".
3. **Real file pull, not acknowledgement.** `versioncon.sync` must call `fsLayer.copyFileToWorkspace(relativePath)` for every non-conflict out-of-sync file. The existing `markSynced` toast text "Workspace files unchanged" must be removed entirely.
4. **Per-file conflict prompt.** For each file where workspace contents differ from the branch contents, show a prompt with three options: **Keep mine**, **Take branch**, **Show diff**. "Show diff" opens `vscode.diff` and re-prompts after the diff editor is closed (the user picks one of the other two after seeing the diff).
5. **Keep mine semantics.** "Keep mine" leaves the file as-is in the workspace and leaves that path in the out-of-sync set. After a sync run where some files chose "Keep mine", the user is partially synced — the status bar warning stays on. Only a subsequent sync (or `Take branch` on those files) clears the state.

When the user picks **Sync** in the debug-start modal, the listener MUST call `vscode.debug.stopDebugging(session)` on the started session before invoking the sync command. Tasks have no equivalent stop API; document this as an accepted v1 limit (the modal still interrupts the user visibly; once the user clicks Sync, the task continues running while sync executes — recommend the user kill it manually if needed).
7. **Removal of markSynced.** All references to `versioncon.markSynced` (extension.ts, package.json, any test) must be removed. Use `grep -c "versioncon.markSynced" src/extension.ts package.json` to verify (must return 0).

The implementation walks `syncTracker.getOutOfSyncPaths()`, partitions each path into one of four cases (deleted-upstream / no-local-copy / identical-bytes / real-conflict), pulls non-conflict files via `fsLayer.copyFileToWorkspace(rel)`, and presents a per-file modal with **Keep mine / Take branch / Show diff** for each real conflict:

    ```typescript
        // --- PUSH-10/11: Sync command ---
        // Real file pull. Walks the SyncTracker out-of-sync set, partitions into:
        //   * identical    -> branch and workspace bytes match -> silent no-op (clear path)
        //   * no-local     -> file exists in branch but not in workspace -> silent copy
        //   * conflict     -> file exists in both, bytes differ -> per-file prompt:
        //                       Keep mine | Take branch | Show diff
        // After the loop, if the out-of-sync set is empty, mark fully synced.
        // If non-empty (the user kept some local edits), keep the warning on.
        context.subscriptions.push(
          vscode.commands.registerCommand('versioncon.sync', async () => {
            const paths = syncTracker.getOutOfSyncPaths();
            if (paths.length === 0) {
              syncTracker.onSync();
              statusBarManager.setSyncWarning(false);
              void vscode.window.showInformationMessage('VersionCon: already in sync.');
              return;
            }

            const branchDir = fsLayer.getBranchDir();
            const projectRoot = fsLayer.getProjectRoot();
            let pulled = 0;
            let kept = 0;

            for (const rel of paths) {
              const branchPath = path.join(branchDir, rel);
              const workspacePath = path.join(projectRoot, rel);

              const branchExists = fs.existsSync(branchPath);
              const workspaceExists = fs.existsSync(workspacePath);

              // Case A: file no longer in branch (deleted upstream).
              // v1: skip the file but clear it from the set so the user is not stuck.
              // (Workspace deletion semantics are intentionally deferred.)
              if (!branchExists) {
                syncTracker.clearPath(rel);
                continue;
              }

              // Case B: workspace has no local copy -> silent pull.
              if (!workspaceExists) {
                await fsLayer.copyFileToWorkspace(rel);
                syncTracker.clearPath(rel);
                pulled++;
                continue;
              }

              // Case C: both exist -> compare bytes.
              const [branchBuf, workspaceBuf] = await Promise.all([
                fsPromises.readFile(branchPath),
                fsPromises.readFile(workspacePath),
              ]);
              if (branchBuf.equals(workspaceBuf)) {
                // Identical -> nothing to lose, silent clear.
                syncTracker.clearPath(rel);
                continue;
              }

              // Case D: real conflict -> PUSH-11 per-file prompt.
              // Loop until the user picks Keep mine or Take branch (Show diff
              // re-prompts after the diff editor is opened).
              let resolved = false;
              while (!resolved) {
                const choice = await vscode.window.showInformationMessage(
                  `Sync conflict: ${rel}`,
                  { modal: true, detail: 'The branch has a different version of this file than your workspace. Choose how to resolve:' },
                  'Keep mine',
                  'Take branch',
                  'Show diff',
                );
                if (choice === 'Take branch') {
                  await fsLayer.copyFileToWorkspace(rel);
                  syncTracker.clearPath(rel);
                  pulled++;
                  resolved = true;
                } else if (choice === 'Keep mine') {
                  // PUSH-11: leave the file as-is in the workspace AND leave it
                  // in the out-of-sync set so the user can come back to it.
                  // Do NOT call clearPath.
                  kept++;
                  resolved = true;
                } else if (choice === 'Show diff') {
                  const branchUri = vscode.Uri.file(branchPath);
                  const workspaceUri = vscode.Uri.file(workspacePath);
                  await vscode.commands.executeCommand(
                    'vscode.diff',
                    branchUri,
                    workspaceUri,
                    `${rel} (Branch ↔ Workspace)`,
                  );
                  // After the diff editor opens, loop and re-prompt.
                  // (User must still pick Keep mine or Take branch.)
                } else {
                  // User dismissed the conflict modal (Esc / X). PUSH-11 says
                  // any unresolved file stays in the out-of-sync set. Treat
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Remote push -> local SyncTracker | Push notifications from other members carry `data.files[].relativePath`. The list is consumed by `recordRemoteFiles` and read back by `versioncon.sync` to drive `fsLayer.copyFileToWorkspace`. |
| Branch directory <-> workspace | Same trust boundary as Phase 2's drag-and-drop pull (UI-04): the branch dir is filesystem-backed and host-controlled. `FileSystemLayer.copyFileToWorkspace` already validates against path traversal (validateBranchPath line 242 in FileSystemLayer.ts). |
| VS Code modal dismiss | User-driven event; the only "exit" is **Sync** or dismiss-as-cancel. No third path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Denial of Service | Repeated modal blocks on rapid pushes spam the user | accept | The modal only fires when `!syncTracker.isInSync()`. Each user action checks once and either proceeds or routes to Sync. After Sync the user is in-sync until the next remote push, so the modal is non-recurring per push event. Multiple rapid pushes accumulate in the file set but only one modal interruption per user action |
| T-03-10 | Tampering | Extension host crash mid-pull leaves workspace in mixed state | accept (v1) | `fsLayer.copyFileToWorkspace` is per-file atomic (`fs.copyFile`) but the loop is not transactional. If the extension crashes during sync, some files may be pulled and others not. The next sync run will discover the remaining out-of-sync files via `getOutOfSyncPaths()` and re-prompt. Documented as a known v1 limit; transactional sync is out of scope for Phase 3 |
| T-03-11 | Tampering / Trust | Branch dir is host-controlled; a malicious host could craft a path-traversal `relativePath` | mitigate | `FileSystemLayer.copyFileToWorkspace` already calls `validateBranchPath(relativePath)` which throws on any path that resolves outside the branch directory (Phase 2, line 242 in FileSystemLayer.ts). No new attack surface introduced by this plan; we reuse the existing validated copy primitive |
| T-03-12 | Spoofing | A `push-received` message with a fabricated pushId could mark the workspace out of sync without a real branch change | accept | Same trust model as Plan 03-04 — the host authenticates members at session-join time and SessionHost is the source of all push broadcasts. A malicious peer cannot craft messages to other peers; messages are relayed by the host (PERMISSION_DENIED relay check in 03-03, SessionHost.ts lines 259-268). Out-of-scope for this plan to harden further |
| T-03-13 | Information Disclosure | Sync command shows file paths in modal that may leak repository structure | accept | The same paths are already visible in the workspace tree, branch tree, push notifications, and push history. No new disclosure |
| T-03-14 | Denial of Service | An extremely large out-of-sync set (hundreds of files) means hundreds of conflict modals | accept (v1) | Worst case: per-file modal loop blocks until user clicks through every conflict. Acceptable v1 — solo-developer-style team workflow rarely produces hundreds of conflicting files in a single sync. Phase 5 dependency-impact filtering will reduce noise; bulk "Take branch all" is a v2 enhancement |
