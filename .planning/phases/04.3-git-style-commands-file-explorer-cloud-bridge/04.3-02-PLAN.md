     vscode.commands.registerCommand('versioncon.pull', async () => {
       const differ = new WorkspaceDiffer();
       const diff = await differ.diff(
         workspaceFolder.uri.fsPath,
         fsLayer.getBranchDir(),
       );
       const candidates = [...diff.added, ...diff.modified];
       if (candidates.length === 0) {
         void vscode.window.showInformationMessage('VersionCon: workspace is already up to date with the branch.');
         return;
       }
       let pulled = 0;
       let kept = 0;
       for (const rel of candidates) {
         const branchPath = path.join(fsLayer.getBranchDir(), rel);
         const workspacePath = path.join(workspaceFolder.uri.fsPath, rel);
         const branchExists = fs.existsSync(branchPath);
         const workspaceExists = fs.existsSync(workspacePath);
         if (!branchExists) continue;
         if (!workspaceExists) {
           await fsLayer.copyFileToWorkspace(rel);
           pulled++;
           continue;
         }
         const [bBuf, wBuf] = await Promise.all([
           fsPromises.readFile(branchPath),
           fsPromises.readFile(workspacePath),

         while (!resolved) {

             `Pull conflict: ${rel}`,

               modal: true,

             'Keep mine',
             'Take branch',
             'Show diff',

             await fsLayer.copyFileToWorkspace(rel);

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workspace filesystem ↔ `.versioncon/branches/{branch}/` | Both sides under user control; risk = accidental data loss during overwrite |
| `versioncon.pull` → existing PUSH-11 prompt | Loop must be re-entrant; risk = infinite loop on a dismissed prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.3-06 | Tampering (data loss) | `vc.pull` overwriting workspace files | mitigate | Byte-compare before copy: if branch and workspace bytes match, skip silently. If different, ALWAYS prompt — never auto-overwrite. Reuse the existing PUSH-11 modal (extension.ts:2244) verbatim — same three options, same loop. Tested by source-grep on the literal strings. |
| T-04.3-07 | Tampering | `vc.push` auto-stage including files the user did not intend (e.g. accidentally created files in workspace) | mitigate | Exclusion list hard-coded for `.versioncon`, `.vscode`, `node_modules`, `dist`, `.git`, `.DS_Store`. Existing confirmation modal (extension.ts:1435) shows the file list before push — user can cancel. Tested by behavior tests 5-6 on exclusion rules. |
| T-04.3-08 | Denial of service | WorkspaceDiffer on a huge tree blocks the event loop | accept | v1: walks files synchronously per call. Most workspaces are <10K files; readFile is async. Pathological case (millions of files) is out of scope for v1 — Phase 5 will add chunked/streaming diff. |
| T-04.3-09 | Information disclosure | `vc.pull` Show-diff opens vscode.diff which displays both file contents | accept | Same as existing sync handler — Show-diff is a documented PUSH-11 affordance and the workspace user owns both files. |
</threat_model>
