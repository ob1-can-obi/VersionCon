# Phase 4.3 Plan 02: Workspace-Diff Push/Pull (no drag required) тАФ Summary

SC-3 + SC-4 delivered. `versioncon.push` now works without drag-and-drop staging тАФ when nothing is staged via the existing split-pane UI, a new `WorkspaceDiffer` service walks workspace-vs-branch, byte-compares each file, and feeds `allChanged` straight into the existing `PushService.executePush` pipeline. A brand-new `versioncon.pull` command does the inverse pull, reusing the PUSH-11 conflict modal verbatim (T-04.3-06 mitigation). Drag-and-drop split-pane is untouched (SPEC scope guardrail).

## What shipped

1. **`WorkspaceDiffer` service.** `src/services/WorkspaceDiffer.ts` (~200 lines). Pure service, two-argument `diff(projectRoot, branchDir, extraIgnoreGlobs?)` returning `{ added, modified, deleted, allChanged }`. All paths POSIX-style relative, sorted lexicographically. Hard-coded exclusion of `.versioncon/`, `.vscode/`, `node_modules/`, `dist/`, `.git/`, `.DS_Store` (T-04.3-07 mitigation). Byte-compare via `Buffer.equals`. Tolerates missing branch or workspace dir.

2. **`versioncon.push` auto-stage fallback (SC-3).** Existing push handler gains a `staged.length === 0` branch that runs `WorkspaceDiffer.diff(...)` against the workspace and the active branch dir; `diff.allChanged` becomes the staged-paths list fed into `PushService.executePush`. `autoStaged` flag prevents the post-push `clearStaged()` from clobbering UI state that was never set. The empty-warning becomes "No workspace changes to push" when the diff itself is empty. Permission gates (locked branch, `canPushToBranch`) run on the resolved `stagedPaths` regardless. Drag-and-drop path preserved verbatim in the `else` branch.

3. **`versioncon.pull` (SC-4).** New dedicated command, registered immediately after `versioncon.sync` inside the workspace IIFE. Computes a fresh diff via the same `WorkspaceDiffer`, takes `added тИк modified` as candidates, and for each: silently copies the branch file when the workspace is missing it, skips on byte-equal, and reuses the PUSH-11 modal verbatim on byte-different conflicts. Branch-side deletions (`diff.deleted`) are NOT propagated in v1 тАФ same scope decision as the sync handler's deleted-upstream behavior; documented inline.

4. **`cmd.pull` alias retarget.** `src/commands/aliases.ts` updated: `versioncon.cmd.pull` now routes to `versioncon.pull` (was `versioncon.sync`). Per the SC-4 contract, pull and sync are now distinct: pull = "give me the latest branch state"; sync = "resolve my out-of-sync tracker set after someone else pushed." Both coexist.

5. **17 new regression tests.** `src/test/suite/workspaceDiff.test.ts` тАФ three suites:
   - `Phase 4.3 Wave 2 тАФ WorkspaceDiffer` (9 behavior tests, tmpdir-backed)
   - `Phase 4.3 Wave 2 тАФ vc push auto-stage fallback (SC-3)` (4 source-grep tests)
   - `Phase 4.3 Wave 2 тАФ vc pull (SC-4)` (4 source-grep tests, including PUSH-11 literal reuse)

## Decisions made

1. **WorkspaceDiffer is a pure service, two absolute paths in.** No `FileSystemLayer` injection. Makes the unit tests trivially scratch-dir-backed and keeps the service reusable for any future "compare these two trees" need (Phase 5 dependency-aware diff is a natural extension that will likely subclass or compose this).

2. **Substring-match `extraIgnoreGlobs`, not full glob.** SPEC scope says "common ignore patterns" and explicitly defers richer ignore semantics. Adding a glob dep (`micromatch`, `picomatch`) for v1 would have brought transitive deps for marginal gain. Future plans can swap the substring check for a glob matcher behind the same parameter without changing callers.

3. **Lexicographic sort applied to each bucket AND to `allChanged`.** Tests assert sorted output explicitly тАФ V8's Array.sort is not stable for equal keys, and any byte-equal "ties" inside `allChanged` (impossible here since paths are unique, but future-proof) need deterministic ordering for test stability across reruns.

4. **`vc.pull` v1 SKIPS branch-side deletions.** Mirrors the sync handler's deleted-upstream behavior (extension.ts ~2343-2346): branch missing тЖТ drop, do not delete workspace file. Inline comment documents the scope decision. Workspace-deletion semantics need a separate UX call (does pulling mean "match the branch exactly" or "give me what's there"?) тАФ out of scope for SC-4 as written.

5. **PUSH-11 modal LITERALS reused verbatim, not via a shared helper.** T-04.3-06 mitigation depends on the user seeing the same wording in both flows so the modal feels predictable. Source-grep tests pin the three option strings ('Keep mine', 'Take branch', 'Show diff') AND the detail wording. A future refactor could extract a helper, but the test contract would still pin the literal strings тАФ extraction is a follow-up.

6. **Confirmation modal switched from `staged.length` to `stagedPaths.length`.** Pre-existing bug-on-auto-stage: when `staged` is empty but we auto-stage 5 files, reporting "Push 0 file(s)" would be a silent UX wart. Rule 1 deviation тАФ the swap is correct for both the drag-staged and auto-staged paths.

7. **`autoStaged` flag gates `clearStaged()` calls.** When auto-staged, there is nothing in the drag-stage set to clear, so the calls are no-ops тАФ but the explicit guard documents intent and survives future refactors that might add side-effects to clearStaged.

- **Found during:** Task 2 тАФ first run of `versioncon.push fallback runs WorkspaceDiffer when nothing is drag-staged` failed because the regex used `[\s\S]{0,400}` between the `if` opener and the `new WorkspaceDiffer()` site. - **Fix:** Widened the byte-caps to `{0,1500}` / `{0,800}` and added a secondary assertion that pins the literal args passed to `differ.diff(...)`. The new shape is more permissive across whitespace changes and still locks the critical wiring.
- **Files modified:** `src/test/suite/workspaceDiff.test.ts`.
- **Commit:** `3b79f6b`.
