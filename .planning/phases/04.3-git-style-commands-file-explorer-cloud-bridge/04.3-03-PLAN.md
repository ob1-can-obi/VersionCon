<objective>
Add a status-bar indicator that surfaces uncommitted workspace changes without the user opening the split-pane (SPEC SC-5). When the workspace has changes relative to the active branch dir, a status-bar item appears reading `$(git-pull-request) N local change(s)`. Clicking it runs `versioncon.diff` — a QuickPick preview of what would be pushed; selecting a file opens vscode.diff between branch and workspace.

Purpose: Phase 4 UAT 2026-05-11 user feedback was that the `.versioncon/branches/` model is invisible and "I never know if I'm out of sync." The status bar makes the state legible at a glance — a discoverable surface for `vc push` without forcing the user into the split-pane.

Output: New `LocalChangesStatusBar` UI class + a debounced FileSystemWatcher + a `versioncon.diff` command. The existing `StatusBarManager` (connection state + unread chat) is unchanged.
</objective>

   ```typescript
   import * as vscode from 'vscode';
   import type { DiffResult } from '../services/WorkspaceDiffer.js';

   /**
    * Phase 4.3 SC-5: status-bar item that surfaces uncommitted workspace
    * changes relative to the active branch dir. Distinct from
    * StatusBarManager (connection state + unread chat) — right-aligned,
    * lower priority, hidden when there are zero changes.
    *
    * Click runs versioncon.diff (Plan 04.3-03 Task 3) which opens a
    * QuickPick of changed files. Tooltip shows the first 5 changed paths.
    */
   export class LocalChangesStatusBar implements vscode.Disposable {
     private readonly item: vscode.StatusBarItem;
     constructor() {
       this.item = vscode.window.createStatusBarItem(
         vscode.StatusBarAlignment.Right,
         99,
       );
       this.item.command = 'versioncon.diff';
     }
     refresh(diff: Pick<DiffResult, 'allChanged'>): void {
       const n = diff.allChanged.length;
       if (n === 0) {
         this.item.hide();
         return;
       }
       const label = n === 1 ? 'local change' : 'local change(s)';
       this.item.text = `$(git-pull-request) ${n} ${label}`;
       const preview = diff.allChanged.slice(0, 5).join('\n');
       const more = n > 5 ? `\n…and ${n - 5} more` : '';
       this.item.tooltip = `Click to preview before pushing\n${preview}${more}`;
       this.item.show();
     }
     hide(): void { this.item.hide(); }
     // Test helpers — same pattern as StatusBarManager.getItemTextForTest()
     getItemTextForTest(): string { return this.item.text; }
     getItemCommandForTest(): string | vscode.Command | undefined { return this.item.command; }
     getItemTooltipForTest(): string | vscode.MarkdownString | undefined { return this.item.tooltip; }
     dispose(): void { this.item.dispose(); }
   }
   ```

   ```typescript
   context.subscriptions.push(
     vscode.commands.registerCommand('versioncon.diff', async () => {
       const differ = new WorkspaceDiffer();
       const diff = await differ.diff(
         workspaceFolder.uri.fsPath,
         fsLayer.getBranchDir(),
       );
       if (diff.allChanged.length === 0) {
         void vscode.window.showInformationMessage('VersionCon: no local changes.');
         return;
       }
       const addedSet = new Set(diff.added);
       const deletedSet = new Set(diff.deleted);
       const items = diff.allChanged.map(rel => {
         const prefix = addedSet.has(rel) ? 'A ' : deletedSet.has(rel) ? 'D ' : 'M ';
         return { label: `${prefix}${rel}`, description: rel, rel };
       });
       const picked = await vscode.window.showQuickPick(items, {
         title: `VersionCon: ${diff.allChanged.length} local change(s)`,
         placeHolder: 'Pick a file to diff against branch',
         matchOnDescription: true,
       });
       if (!picked) return;
       const branchPath = path.join(fsLayer.getBranchDir(), picked.rel);
       const workspacePath = path.join(workspaceFolder.uri.fsPath, picked.rel);
       await vscode.commands.executeCommand(
         'vscode.diff',
         vscode.Uri.file(branchPath),
         vscode.Uri.file(workspacePath),
         `${picked.rel} (Branch ↔ Workspace)`,
       );
     }),
   );
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FileSystemWatcher → WorkspaceDiffer (recomputed every 500ms on activity) | Performance, not security |
| versioncon.diff QuickPick → vscode.diff command | The QuickPick label could leak file paths if the workspace has sensitive filenames; mitigated by the user explicitly clicking the status-bar item |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.3-10 | Denial of service | Workspace-wide watcher fires on every save in node_modules / dist / etc., causing CPU thrash | mitigate | WorkspaceDiffer's exclusion list (DEFAULT_IGNORE_DIRS) skips those dirs inside the diff walk. Watcher itself fires on the file event, but the diff walk dominates and is bounded by exclusion. 500ms debounce coalesces rapid events. If pathological hangs occur in real use, Phase 5 can switch the watcher glob to an inclusion list. |
| T-04.3-11 | Information disclosure | QuickPick lists all changed files including hidden ones | accept | User initiated the action (clicked the status-bar); paths shown are workspace-relative POSIX (no absolute paths exposed). Same disclosure level as existing pushHistory modal. |
| T-04.3-12 | Tampering | Race condition: file save → watcher fires → diff reads file before save completes → wrong N | accept | 500ms debounce gives the FS time to settle. If a save is still in-flight when the debounce fires, the next event will retrigger and produce a correct count within the next 500ms window. Self-healing. |
</threat_model>
