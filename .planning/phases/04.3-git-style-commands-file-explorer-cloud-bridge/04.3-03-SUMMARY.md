# Phase 4.3 Plan 03: Local-Changes Status-Bar Indicator + `versioncon.diff` QuickPick — Summary

SC-5 delivered. The workspace now grows a second status-bar item that surfaces uncommitted local changes within 500ms of a file save — a discoverable surface for `vc push` that does NOT require opening the split-pane. Click the item, get a QuickPick of changed files; pick one, get the branch ↔ workspace diff view. The existing `StatusBarManager` (connection state + unread chat overlay) is untouched — the new indicator is a sibling, not a replacement.

## What shipped

1. **`LocalChangesStatusBar` UI class.** `src/ui/LocalChangesStatusBar.ts` (~110 lines). Right-aligned StatusBarItem with priority 99 — opposite side of the status bar from `StatusBarManager` (Left, priority 100). Click command preset to `versioncon.diff` at construction. `refresh(diff)` renders `$(git-pull-request) N local change(s)` (singular for N=1) and shows the item when `diff.allChanged.length > 0`; hides it when empty. Tooltip lists the first 5 changed paths with a `…and X more` overflow suffix. Test helpers (`getItemTextForTest`, `getItemCommandForTest`, `getItemTooltipForTest`) mirror the `StatusBarManager.ts:151-160` pattern verbatim.

2. **Workspace-wide FileSystemWatcher + 500ms debounce.** Inside the workspace IIFE in `src/extension.ts`, a NEW `wsWatcher` is registered on `RelativePattern(workspaceFolder, '**/*')` — distinct from the existing branchTree watcher (root `.versioncon/branches/**/*`, debounce 50ms, consumer `branchProvider.refresh()`). The new watcher's three event handlers (onDidCreate / onDidChange / onDidDelete) all share `recomputeLocalChanges`, which clears any pending timer and sets a fresh `setTimeout(..., 500)`. Inside the timer the WorkspaceDiffer runs, its `DiffResult` flows into `localChangesStatusBar.refresh(diff)`. A one-shot `recomputeLocalChanges()` runs after registration so the bar reflects state at activation time. Diff failures are caught and console.error'd — they MUST never break the watcher loop.

3. **`versioncon.diff` QuickPick command.** Registered programmatically inside the workspace IIFE (NOT in `package.json` — the entry points are the status-bar click and the `cmd.diff` alias, per the plan's DO-NOT-DO list). Runs a fresh `WorkspaceDiffer.diff` against the active branch dir, builds QuickPick items labeled `A path` / `M path` / `D path` (git-style status prefixes), and on pick invokes `vscode.diff` with `Uri.file(branchPath)` ↔ `Uri.file(workspacePath)` and a `${rel} (Branch ↔ Workspace)` title — mirrors the existing per-file `versioncon.previewDiff` at extension.ts:1480-1484. Empty diff emits an information message and returns; no QuickPick shown.

4. **`cmd.diff` alias retarget.** `src/commands/aliases.ts` updated: `versioncon.cmd.diff` now routes to `versioncon.diff` (was `versioncon.previewDiff`). The per-file `versioncon.previewDiff` command remains its own registered command for the workspaceTree context menu — both commands coexist. `filesExclude.test.ts:285` Wave 1 mapping assertion updated to follow the new target (same pattern as the Wave 2 update for `cmd.pull`).

5. **Session gate (Rule 2 deviation).** The plan refreshed the indicator unconditionally. The prompt-level constraint mandated tying show/hide to `activeHost !== null || activeClient !== null`. `recomputeLocalChanges` now short-circuits to `localChangesStatusBar.hide()` when both are null, BEFORE running the diff — saves a walk too. Source-grep test pins the guard.

6. **19 new regression tests** in `src/test/suite/localChangesStatusBar.test.ts`:
   - `Phase 4.3 Wave 3 — LocalChangesStatusBar (SC-5)` — 7 behavior tests (constructor, N>1, N=1 singular, N=0 hide, tooltip preview + overflow, hide-idempotent, dispose-idempotent).
   - `Phase 4.3 Wave 3 — extension wiring (SC-5)` — 7 source-grep tests (import, singleton + subscriptions push, `**/*` glob, 500ms literal + localChangesDebounce identifier, three onDid* registrations, session gate, initial-refresh ordering).
   - `Phase 4.3 Wave 3 — versioncon.diff QuickPick (SC-5)` — 5 source-grep tests (registration, QuickPick build, vscode.diff on pick, empty-changes guard, cmd.diff alias retarget).

## Decisions made

1. **LocalChangesStatusBar is diff-data-driven, not differ-aware.** `refresh(diff)` takes a `Pick<DiffResult, 'allChanged'>` argument — the class never constructs a `WorkspaceDiffer` or subscribes to a watcher. Keeps the unit tests trivially behavioral (no I/O) and lets the extension.ts wiring own the timing concerns (watcher / debounce / session gate / one-shot init). Single responsibility.

2. **Right alignment + priority 99 — opposite side from StatusBarManager.** No layout collision; the two items appear on opposite ends of the status bar. The Wave 3 plan was explicit that StatusBarManager must NOT be modified — keeping concerns physically separated reinforces that contract.

3. **Click command preset at construction.** `this.item.command = 'versioncon.diff'` in the constructor body, never mutated. StatusBarManager's `applyUnreadOverlay` swaps the click command between `versioncon.showSidebar` and `versioncon.openChat`; LocalChangesStatusBar has exactly one click behavior, so the constructor-time set is sufficient. Simpler reasoning, fewer state transitions to test.

4. **Singular vs plural label.** `n === 1 ? 'local change' : 'local change(s)'`. Mirrors StatusBarManager.flashNoImpact's `'1 file unaffected'` vs `'N file(s) unaffected'` split (StatusBarManager.ts:109-111). Test pins both forms.

5. **Tooltip preview bounded at 5 paths + overflow suffix.** Tooltip growth is bounded so a workspace with 200 changed files doesn't render a wall of text; the user can always click through to the QuickPick to see the full list. The 5-path cutoff is the same shape as VS Code's built-in git extension tooltip.

6. **Workspace-wide watcher uses bare `**/*` glob.** WorkspaceDiffer's `DEFAULT_IGNORE_DIRS` already filters `.versioncon/`, `.vscode/`, `node_modules/`, `dist/`, `.git/` inside the walk (line 34-40 of WorkspaceDiffer.ts). Encoding the same exclusions into the watcher glob would be redundant and create a second filter that could drift from the differ's. The watcher fires more often than strictly needed; the diff walk dominates the cost and is bounded by the exclusion list. T-04.3-10 mitigation strategy (per the plan's threat register).

7. **500ms debounce literal pinned via source-grep test.** SC-5 mandates "within 500ms of a workspace file save". The source-grep test asserts the literal `, 500)` appears in a setTimeout that wraps a WorkspaceDiffer call. Any future plan that wants a different value MUST update both the code and the test — deliberate friction so the spec stays authoritative.

8. **One-shot `recomputeLocalChanges()` at end of IIFE.** Without it, the status bar only appears AFTER the first FS event post-activation. That violates SC-5's "appears within 500ms of a workspace save" intent for the first-render case — the user might already have changes on disk at activation time. The initial call rounds out the contract.

9. **Session gate (Rule 2 deviation).** Plan did not include this; prompt-level constraint did. Beyond-plan UX correctness — when there's no host or client active, "N local changes" has no peer to push to and the badge is meaningless. The null check runs BEFORE the WorkspaceDiffer construction so we also save the diff walk in the no-session case. Source-grep test pins the guard so a future refactor that drops the null check trips the suite.

10. **`versioncon.diff` registered ONLY programmatically.** Not in `package.json` `contributes.commands`. The plan's DO-NOT-DO list was explicit: user-facing entry points are the status-bar click and the `cmd.diff` alias; Command Palette discoverability is a deliberate non-goal. Avoids confusing users with two diff commands ("VersionCon: Diff" workspace-wide + "VersionCon: Diff" per-file) in the Palette.

11. **`versioncon.previewDiff` preserved verbatim.** Both `versioncon.previewDiff` (per-file from workspaceTree context menu) and the new `versioncon.diff` (workspace-wide QuickPick) coexist. Source-grep test asserts both registrations are present. `cmd.diff` alias was the only thing that needed retargeting — previewDiff's own consumers (the context menu) are unaffected.

12. **Git-style status prefixes in the QuickPick.** `'A path'` / `'M path'` / `'D path'` — familiar to git users (the plan's overarching audience). The description field carries the bare relative path so `matchOnDescription: true` lets users filter on the path without the prefix getting in the way.

13. **`vscode.diff` with `Uri.file(branchPath)` even when branchPath doesn't exist.** Added files have no branch counterpart, but VS Code renders the missing side as an empty document — desired UX (user sees the full added content on the right side). Mirrors the existing per-file previewDiff command at extension.ts:1480-1484 verbatim. The plan's deviation_protocol allowed using `Uri.parse('untitled:')` for added files; the cleaner default behavior of vscode.diff makes that complexity unnecessary.

## Deviations from plan

### Auto-fixed Issues

**1. [Rule 2 — beyond-plan UX guard] Session gate for the indicator**
- **Found during:** prompt-level constraint reading — "indicator should NOT show when there's no active VersionCon session... tie show/hide to `activeHost !== null || activeClient !== null`."
- **Issue:** The plan as written refreshes the indicator unconditionally. A user with `.versioncon/branches/{x}/` files present but no active session would see a stale "N local changes" badge with no peer to push to — actionable-meaning-free UX noise.
- **Fix:** Added a null-check at the top of the `recomputeLocalChanges` setTimeout callback (`if (activeHost === null && activeClient === null) { localChangesStatusBar?.hide(); return; }`). Runs BEFORE WorkspaceDiffer is constructed so it also saves the diff walk. New source-grep test pins the guard literal.
- **Files modified:** `src/extension.ts`, `src/test/suite/localChangesStatusBar.test.ts`.
- **Commit:** `08e6f19`.

**2. [Rule 1 — test regex byte-cap tune] Debounce test cap too tight after Task 2**
- **Found during:** Task 2 first test run after wiring extension.ts. The plan's regex `setTimeout\([\s\S]{0,400}?,\s*500\)` was too tight to span the multiline `try/catch` + IIFE body.
- **Issue:** The actual distance from `WorkspaceDiffer` to `, 500)` is ~590 bytes (because the inside of the setTimeout has a multi-line `void (async () => { try { ... } catch { ... } })()` wrapper for error isolation).
- **Fix:** Bumped the byte cap from 400 to 1500 (post-WorkspaceDiffer) and added a defense-in-depth `localChangesDebounce = setTimeout(` identifier assertion so a refactor that drops the named variable still trips the suite.
- **Files modified:** `src/test/suite/localChangesStatusBar.test.ts`.
- **Commit:** `9820699`.

**3. [Rule 1 — test regex byte-cap tune AGAIN] Debounce pre-cap too tight after session gate**
- **Found during:** Rule 2 deviation commit prep. The session-gate `if (activeHost === null && activeClient === null) { ... }` block added ~600 chars BETWEEN `setTimeout(` and `WorkspaceDiffer`. The post-Task-2 regex `setTimeout\([\s\S]{0,800}?WorkspaceDiffer` no longer matched (actual distance is now 846).
- **Issue:** Without widening, the Rule 2 commit would have re-broken the test we just made GREEN.
- **Fix:** Bumped the leading byte cap from 800 to 2000. The 2000 ceiling is generous enough to absorb future inline comment / log additions in the same block without becoming a "matches any setTimeout in the file" wildcard.
- **Files modified:** `src/test/suite/localChangesStatusBar.test.ts`.
- **Commit:** `08e6f19`.

**4. [Rule 1 — test update] Wave 1 cmd.diff mapping assertion follows the retarget**
- **Found during:** Task 3 — the Wave 1 `filesExclude.test.ts:285` asserted `cmd.diff → versioncon.previewDiff`. Wave 3 explicitly retargets cmd.diff to `versioncon.diff`, so the existing test must follow.
- **Issue:** Existing Wave 1 assertion would have failed after my aliases.ts edit.
- **Fix:** Updated the mapping table entry to the new target with an inline comment explaining the Wave 3 retarget. Mirrors the Wave 2 update for cmd.pull at line 278-281.
- **Files modified:** `src/test/suite/filesExclude.test.ts`.
- **Commit:** `276bd91`.
