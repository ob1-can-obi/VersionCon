   /** Module-private registry. Wave 2 wires JS/TS adapters; Wave 3 wires Python + fallback stubs for Java/C++. */

   export function _resetRegistryForTests(): void {
     REGISTRY.clear();

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | DoS | skipPolicy file-size gate | mitigate | Wave 1 enforces DEFAULT_MAX_FILE_BYTES = 500_000 at the contract level. Wave 4 worker stats every file BEFORE reading content and calls shouldSkip(rel, size) — if true, the file is excluded from the AnalyzePayload, never crosses IPC. Wave 5 host-side also gates pre-IPC. Tests pin the cap value so a future refactor cannot silently raise it. |
| T-05-01 | DoS via worker crash | AstAdapter interface contract | mitigate (deferred to Wave 4) | Wave 1 declares AstAdapter as `(source, relativePath) → SymbolIndex/ReferenceIndex` — synchronous methods that adapters MUST wrap their tree-sitter parse calls inside. Wave 4 owns the parent-side re-fork-on-exit; Wave 1's role is to make the contract crash-tolerant by NOT requiring adapters to be async (avoid timer leaks) and by NOT exposing tree-sitter types in the public interface. |
| T-05-02 | Slowloris parse | AstAdapter interface contract | mitigate (deferred to Wave 4) | Wave 1's AstAdapter methods are pure synchronous `(source, relativePath) → Index` — Wave 4 wraps each call in a 5s child-process-side timeout. Wave 1 cannot enforce the timeout itself because Promise.race in the host process would consume the extension-host thread (defeats SC-2). |
| T-05-03 | Path escape in worker IPC | AnalyzePayload interface | mitigate (deferred to Wave 4) | Wave 1 declares AnalyzePayload's `changedFiles[].relativePath` as `string` — Wave 4 worker MUST validate each path is under the workspace root + branch dir using the same segment-aware traversal check as Plan 04-15 (`segments.includes('..')`). Wave 1's IPC frame design avoids absolute paths entirely; only workspace-relative paths cross IPC. |
| T-05-05 | Wrong-attribution (symbol name collision) | AffectedSymbol shape | mitigate | Wave 1 defines AffectedSymbol with `changedIn: string` (file path) AND `callers[].file: string` — Wave 4 join logic MUST match on `name AND callers.file ==-or-imports changedIn`. Cross-file scope-aware matching deferred to 5.x per SPEC scope guardrails. The shape itself encodes the file-scoping invariant. |
