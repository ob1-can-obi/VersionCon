   - `postUri = vscode.Uri.file(path.join(versionconDir, 'branches', branchName, relativePath))` — current branch content.
   - Title = `Review: {relativePath}`.
   - If `preUri` does not exist on disk (newly added file in the push — `saveSnapshot` was not called for it because branchExists was false in PushService.ts:117), use `vscode.Uri.parse('untitled:Empty (new file)')` or a tempfile with empty content. Match existing behavior in `versioncon.previewDiff` (PushService.ts caller; check src/extension.ts for the previewDiff command — same edge case handled there).

   const wireFrame: ProtocolMessage = {

Both paths funnel into the SessionHost onmessage handler from Plan 06-02 for identity override + persistence + system event.

8. **Comment thread display (line-grouped)** — comments rendered inline in the panel, grouped by `{filePath}:{line}`. Wave 5 owns the per-line vscode.commentController integration — this plan owns the THREAD VIEW inside the webview panel. Comment input wiring (a "Reply" form per group) posts `{type:'review-comment-submit', reviewId, filePath, line, body}`. ReviewPanel.onDidReceiveMessage constructs a wire frame analogous to the vote path.

import type { ReviewRequest, ReviewComment, ReviewVoteRecord } from '../types/review.js';

import type { PushRecord } from '../types/push.js';

/**
 * Phase 6 (Plan 06-04): WebviewPanel singleton scoped to a specific PushRecord.
 * Architecture mirrors ChatPanel (Plan 04-10) — same CSP shape, same nonce
 * pattern, same dispose discipline, same fallback HTML for missing dist/.
 *
 * Lifecycle:
 *   - createOrShow(context, pushId, ...) — opens new panel OR refreshes existing
 *     scoped to the same pushId. If a panel exists for a DIFFERENT pushId, it
 *     is closed first (singleton invariant; one open review at a time keeps
 *     the UI mental model simple).
 *   - refresh(state) — pulls the current review for this.scopedPushId from the
 *     supplied ReviewState and posts a {type:'state', ...} message to the
 *     webview.
 *   - dispose() — removes all listeners, disposes the panel; static currentPanel
 *     is cleared.
 *
 * Message routing (webview → extension.ts):
 *   - webview-ready          → ReviewPanel posts initial state
 *   - open-file-diff         → ReviewPanel calls vscode.commands.executeCommand('vscode.diff', ...)
 *   - review-vote-submit     → ReviewPanel forwards to onVoteRequested callback
 *   - review-comment-submit  → ReviewPanel forwards to onCommentRequested callback
 *   - review-resolve-submit  → ReviewPanel forwards to onResolveRequested callback
 *
 * extension.ts wires the callbacks at construction. Callbacks are responsible
 * for routing the action onto the wire (SessionClient.sendMessage for joiners,
 * SessionHost.handleLocalReviewVote for the host).
 */
export class ReviewPanel {
  public static currentPanel: ReviewPanel | undefined;
  public readonly scopedPushId: string;
  private readonly panel: vscode.WebviewPanel;
  private readonly context: vscode.ExtensionContext;
  private readonly disposables: vscode.Disposable[] = [];
  private readonly versionconDir: string;
  // Callbacks set by createOrShow caller
  private readonly onVoteRequested: (reviewId: string, vote: 'approved'|'changes-requested'|'commented') => void;
  private readonly onCommentRequested: (reviewId: string, filePath: string, line: number, body: string) => void;
  private readonly onResolveRequested: (reviewId: string, resolvedReason: 'merged'|'abandoned') => void;
  // self identity for visibility gates
  private readonly getSelfMemberId: () => string;
  private readonly getHostMemberId: () => string;

  private constructor(
    context: vscode.ExtensionContext,
    pushId: string,
    versionconDir: string,
    callbacks: {
      onVoteRequested: ReviewPanel['onVoteRequested'];
      onCommentRequested: ReviewPanel['onCommentRequested'];
      onResolveRequested: ReviewPanel['onResolveRequested'];
      getSelfMemberId: ReviewPanel['getSelfMemberId'];
      getHostMemberId: ReviewPanel['getHostMemberId'];
    },
  ) {
    this.context = context;
    this.scopedPushId = pushId;
    this.versionconDir = versionconDir;
    this.onVoteRequested = callbacks.onVoteRequested;
    this.onCommentRequested = callbacks.onCommentRequested;
    this.onResolveRequested = callbacks.onResolveRequested;
    this.getSelfMemberId = callbacks.getSelfMemberId;
    this.getHostMemberId = callbacks.getHostMemberId;

    this.panel = vscode.window.createWebviewPanel(
      'versioncon.review',
      `Review: ${pushId.substring(0, 7)}`,
      vscode.ViewColumn.Beside,
      {
        enableScripts: true,
        retainContextWhenHidden: true,
        localResourceRoots: [vscode.Uri.joinPath(context.extensionUri, 'dist', 'webview', 'review')],
      },
    );

    this.panel.webview.html = this.getWebviewContent();

    this.panel.webview.onDidReceiveMessage(
      (msg) => this.handleMessage(msg),
      undefined,
      this.disposables,
    );
    this.panel.onDidDispose(() => this.dispose(), null, this.disposables);
  }

  static createOrShow(
    context: vscode.ExtensionContext,
    pushId: string,
    versionconDir: string,
    callbacks: ConstructorParameters<typeof ReviewPanel>[3],
  ): ReviewPanel {
    // Singleton: dispose any panel scoped to a different pushId.
    if (ReviewPanel.currentPanel && ReviewPanel.currentPanel.scopedPushId !== pushId) {
      ReviewPanel.currentPanel.dispose();
    }
    if (!ReviewPanel.currentPanel) {
      ReviewPanel.currentPanel = new ReviewPanel(context, pushId, versionconDir, callbacks);
    } else {
      ReviewPanel.currentPanel.panel.reveal();
    }
    return ReviewPanel.currentPanel;
  }

  /**
   * Pull the current ReviewRequest for this.scopedPushId from the supplied
   * ReviewState + PushRecord for the file list, then post the full state
   * to the webview. Called after every applyOpened/applyComment/applyVote/
   * applyResolved/applyStateSync that touches a review with matching pushId.
   */
  refresh(reviewState: ReviewState, push: PushRecord | undefined): void {
    const review = reviewState.getActiveReviewForPush(this.scopedPushId)
      ?? reviewState.getReviewByPushId(this.scopedPushId);
    this.panel.webview.postMessage({
      type: 'state',
      review: review ?? null,
      push: push ?? null,
      selfMemberId: this.getSelfMemberId(),
      hostMemberId: this.getHostMemberId(),
    });
  }

  private async handleMessage(msg: unknown): Promise<void> {
    if (typeof msg !== 'object' || msg === null) return;
    const m = msg as Record<string, unknown>;
    switch (m.type) {
      case 'webview-ready':
        // extension.ts will call refresh() right after construction; nothing
        // to do here. (The webview-ready event still serves as a debug signal
        // — extension.ts may listen and log.)
        break;

      case 'open-file-diff':
        if (typeof m.filePath !== 'string') return;
        await this.openFileDiff(m.filePath);
        break;

      case 'review-vote-submit':
        if (typeof m.reviewId !== 'string') return;
        if (m.vote !== 'approved' && m.vote !== 'changes-requested' && m.vote !== 'commented') return;
        this.onVoteRequested(m.reviewId, m.vote);
        break;

      case 'review-comment-submit':
        if (typeof m.reviewId !== 'string' || typeof m.filePath !== 'string') return;
        if (typeof m.line !== 'number' || !Number.isInteger(m.line) || m.line < 1) return;
        if (typeof m.body !== 'string' || m.body.length === 0 || m.body.length > 16_384) return;
        this.onCommentRequested(m.reviewId, m.filePath, m.line, m.body);
        break;

      case 'review-resolve-submit':
        if (typeof m.reviewId !== 'string') return;
        if (m.resolvedReason !== 'merged' && m.resolvedReason !== 'abandoned') return;
        this.onResolveRequested(m.reviewId, m.resolvedReason);
        break;
    }
  }

  /**
   * Invoke vscode.diff against (pre-push snapshot ↔ post-push branch content)
   * for the requested file. Per 06-SPEC.md locked decision: pre-push snapshot
   * already exists in .versioncon/push-snapshots/{pushId}/{relPath} (Phase 3
   * PushHistory). If the snapshot does not exist (newly-added file in this
   * push), use a tempfile or untitled URI with empty content as the pre side.
   */
  private async openFileDiff(relativePath: string): Promise<void> {
    // Trust the post path comes from the active branch dir of the active
    // session (extension.ts ReviewPanel construction passes versionconDir +
    // the branch is implicit via PushRecord.branch — but ReviewPanel doesn't
    // currently know the branch. Read PushRecord.branch from a cached
    // post-refresh state OR pass branchName via the constructor; v1 reads
    // it from a private field this.scopedBranch set during refresh()).
    // For Task 2: store scopedBranch on first refresh; if null, abort with
    // a toast.
    if (!this.scopedBranch) {
      void vscode.window.showWarningMessage('Review state not loaded yet — try again in a moment.');
      return;
    }
    const preFsPath = path.join(this.versionconDir, 'push-snapshots', this.scopedPushId, relativePath);
    const postFsPath = path.join(this.versionconDir, 'branches', this.scopedBranch, relativePath);
    let preUri: vscode.Uri;
    try {
      await fs.promises.access(preFsPath);
      preUri = vscode.Uri.file(preFsPath);
    } catch {
      // No snapshot = newly-added file in this push (PushService.ts:115-118
      // only snapshots when branchExists). Use an untitled empty-doc URI as
      // the pre side so the diff renders ('' vs new file content).
      preUri = vscode.Uri.parse(`untitled:${relativePath}.empty`);
    }
    const postUri = vscode.Uri.file(postFsPath);
    await vscode.commands.executeCommand(
      'vscode.diff',
      preUri,
      postUri,
      `Review: ${relativePath}`,
    );
  }

  /** Scoped branch is set on first refresh(). Required by openFileDiff. */
  private scopedBranch: string | null = null;
  setScopedBranch(branch: string): void {
    this.scopedBranch = branch;
  }

  dispose(): void {
    ReviewPanel.currentPanel = undefined;
    while (this.disposables.length) {
      const d = this.disposables.pop();
      try { d?.dispose(); } catch { /* ignore */ }
    }
    try { this.panel.dispose(); } catch { /* already disposed */ }
  }

  private getWebviewContent(): string {
    const webview = this.panel.webview;
    const nonce = crypto.randomBytes(16).toString('base64');
    const cssUri = webview.asWebviewUri(vscode.Uri.joinPath(this.context.extensionUri, 'dist', 'webview', 'review', 'main.css'));
    const jsUri = webview.asWebviewUri(vscode.Uri.joinPath(this.context.extensionUri, 'dist', 'webview', 'review', 'main.js'));
    const codiconCssUri = webview.asWebviewUri(vscode.Uri.joinPath(this.context.extensionUri, 'dist', 'webview', 'review', 'codicon', 'codicon.css'));
    const csp =
      `default-src 'none'; ` +
      `img-src ${webview.cspSource} data:; ` +
      `style-src ${webview.cspSource} 'nonce-${nonce}'; ` +
      `font-src ${webview.cspSource}; ` +
      `script-src 'nonce-${nonce}';`;
    const templatePath = path.join(this.context.extensionUri.fsPath, 'dist', 'webview', 'review', 'index.html');
    let html: string;
    try {
      html = fs.readFileSync(templatePath, 'utf-8');
    } catch {
      html = [
        '<!DOCTYPE html><html lang="en"><head>',
        '<meta charset="UTF-8">',
        '<meta http-equiv="Content-Security-Policy" content="%%CSP%%">',
        '<link href="%%CODICON_CSS_URI%%" rel="stylesheet">',
        '<link href="%%CSS_URI%%" rel="stylesheet">',
        '</head><body>',
        '<div id="review-root"></div>',
        '<script nonce="%%NONCE%%" src="%%JS_URI%%"></script>',
        '</body></html>',
      ].join('');
    }
    return html
      .replace(/%%CSP%%/g, csp)
      .replace(/%%NONCE%%/g, nonce)
      .replace(/%%CSS_URI%%/g, cssUri.toString())
      .replace(/%%JS_URI%%/g, jsUri.toString())
      .replace(/%%CODICON_CSS_URI%%/g, codiconCssUri.toString());
  }
}
```

|-----------|----------|-----------|-------------|-----------------|
| T-06-02 | Tampering (XSS) | markdown-it rendering in webview | mitigate | OWNER. markdown-it `html: false` config in src/ui/webview/review/main.ts (source-grep test pins). CSP `script-src 'nonce-X'` blocks inline scripts. No remote sources. Mirrors Plan 04-10's bundle config. The body is host-trusted (Plan 06-02 validates length + non-empty) but XSS escape lives at the renderer. |
| T-06-01 | Spoofing | Webview-originated review-vote/comment/resolve actions | mitigate | The webview's onVoteRequested callback constructs a wire frame with EMPTY reviewerMemberId/displayName — the host (Plan 06-02 OWNER) overrides at relay. Even if a malicious webview supplied a non-empty value, the host overrides unconditionally. Defense-in-depth: ReviewPanel.handleMessage validates the message shape (vote-string membership, line bounds, body length) before forwarding to extension.ts. |
| T-06-04 | Tampering (path) | versioncon.diff URIs | mitigate | preFsPath / postFsPath are constructed from `this.scopedPushId` (host-stamped at review-opened) + relative path from PushRecord.files[].relativePath (host-stamped at push time). No user-supplied path enters the URI construction. The path concatenation uses `path.join` to normalize separators. |
| T-06-05 | Spoofing (state replay) | Refresh from ReviewState | mitigate | ReviewPanel reads from ReviewState which is fed only by SessionClient's host-ws handler. There is no peer-to-peer path for review state to enter the panel. |
| T-06-03 | Denial of Service | Webview comment spam | mitigate | Plan 06-02 OWNER enforces 30/min + 500-cap. ReviewPanel.handleMessage clamps body length at 16 KiB defensively before forwarding. The webview's send button can also debounce (Wave 4 UX) but the trust boundary is the host. |
</threat_model>
