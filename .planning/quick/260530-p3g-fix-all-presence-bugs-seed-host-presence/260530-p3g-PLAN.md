<objective>
Fix the two Presence bugs surfaced during Phase 7 two-machine cloud UAT (UAT-3b):

**Bug 1 — host missing from a joiner's Presence panel.** The host's own presence
only enters `SessionHost.presenceMap` if `extension.ts broadcastSelfPresenceOnJoin`
fired with `activeHost` + `currentSelfMemberId` already set. When the host idles on
the Welcome tab, `onDidChangeActiveTextEditor` never fires, so that seeding is
timing-fragile / may never run and `getPresenceSnapshot()` is empty of the host →
joiners see only themselves. Fix: seed the host's own presence DETERMINISTICALLY in
`SessionHost.handleAuthRequest` (role==='host' branch, after `this.hostMemberId` is
set, before `this.members.set`) so every joiner's snapshot includes the host
regardless of editor activity.

**Bug 2 — duplicate member entries for one joiner.** `handleAuthRequest` mints a
fresh `crypto.randomUUID()` per auth with NO stable client identity, so each
reconnect / cloud bootstrap-swap / repeat-join creates a NEW member entry and stale
entries are not reliably pruned in cloud mode. Fix: introduce an OPTIONAL stable
`clientId` — the joiner generates it ONCE per join (stable across ReconnectManager
reconnects) and sends it in the auth-request; the host dedupes by clientId, rebinding
the connection and reusing the same memberId instead of inserting a second member.
Missing clientId = legacy → today's behavior, zero regression. The host-loopback
connection now ALSO carries a stable clientId (Task 1), so the dedupe block is
explicitly SKIPPED for role==='host' — a host reconnect must always re-run the
role-check + member registration + Task-3 host-presence seed.

Purpose: Presence panel mirrors the live session roster on both LAN and cloud.
Output: protocol.ts + SessionClient.ts + SessionHost.ts edits, all TDD-covered.

CONSTRAINTS (CONTEXT-locked): NO periodic presence heartbeat — keep event-driven.
Preserve LAN/cloud envelope byte-compatibility for existing frames. Surgical,
well-isolated edits. Atomic commits, normal messages, NO Co-Authored-By trailers
(per global rule + CLAUDE.md).
</objective>
